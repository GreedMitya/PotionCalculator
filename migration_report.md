# Отчет о модернизации проекта (Апрель 2026)

## Что было сделано

В ходе очистки проекта и приведения его к современным стандартам (KMP, Compose Multiplatform на 2026 год) были успешно выполнены следующие задачи:

### 1. Внедрение Version Catalogs 📦
Полностью удалены хардкод-версии из `composeApp/build.gradle.kts`. Все зависимости (включая Ktor, DataStore, Lifecycle ViewModel, Splashscreen, Material) были прописаны и вынесены в `gradle/libs.versions.toml`. Это устранит конфликты версий и упростит их дальнейшую поддержку:

> [!TIP]
> Теперь вы можете управлять **всеми** библиотеками в одном месте — `libs.versions.toml`. Это стандарт де-факто для KMP.

### 2. Зачистка устаревшего API (Accompanist) 🧹
Библиотека `accompanist-systemuicontroller`, которая давно признана устаревшей (deprecated) и больше не поддерживается Google, была **удалена** из проекта. 
В файле `AdnroidApp.kt` больше не используется костыль для управления статус барами, так как приложение уже корректно управляет ими нативно через `enableEdgeToEdge()` / `WindowInsetsControllerCompat` в `MainActivity.kt`.

### 3. Обновление до актуальных технологий 🚀
Произведено обновление ключевых библиотек:
*   **Ktor** обновлен с легаси `2.3.5` ➡️ до актуальной стабильной ветки `3.0.0`. Это даст лучшую производительность сети за счет нового движка `kotlinx-io`.
*   **Kotlinx Serialization** обновлена с отстающей `1.6.0` ➡️ на современную `1.7.3`, полностью совместимую с вашим Kotlin `2.1.21`.
*   **DataStore Preferences** обновлен до версии `1.1.1` с поддержкой Kotlin Multiplatform (чтобы при добавлении iOS модуля можно было использовать DataStore в `commonMain`).

### 4. Повышение версии Android SDK ⬆️
`compileSdk` и `targetSdk` были подняты до **36**. Теперь проект полностью готов к новейшим возможностям Android.

## Верификация
После переезда я выполнил команду двойной проверки:
```bash
./gradlew assembleDebug
```
Сборка завершилась **безупречно (Exit code: 0)**! Ваше приложение полностью готово к дальнейшей разработке и соответствует лучшим практикам 2026 года.

> Все детали об изменениях можно посмотреть в git diff-ах файлов `libs.versions.toml` и `composeApp/build.gradle.kts`
